Resurrecting E-Waste for Klipper

Turning old hardware into a powerful 3D printer host
By Marius Måle Stokken

Getting Started

1
Prepare Your Hardware
Hardware selection and OS prep

Don't throw away that old laptop! Any x86 machine with at least 2GB of RAM is more than enough to run Klipper. The biggest bottleneck in old hardware is usually a failing mechanical hard drive.

ℹ️ The SSD Advantage

Swapping a dying HDD for a cheap SSD makes the OS feel modern and snappy. It also eliminates the reliability issues common with SD cards used in Raspberry Pis.

⚠️ BIOS/UEFI Settings

Disable "Secure Boot" in your BIOS if the Debian installer struggles to boot. Look for "AC Auto-On" or "Wake-on-AC" settings if you want the laptop to turn on automatically when power is restored.

2
Install Debian Server
A lean, headless foundation

Flash the Debian Netinst ISO to a USB drive using BalenaEtcher or Raspberry Pi Imager. During the installation process:

  • Deselect Desktop Environments: Keep it "headless" to save system resources.
  • Select SSH Server: Ensure this is checked to manage the machine remotely.
  • Set up User: Create your standard user account.

💡 Sudo Privileges

If you forgot to add your user to the sudo group, log in as root and run: usermod -aG sudo [your_username]. You must reboot for this to take effect.

3
Deploying Klipper via KIAUH
The easiest way to get up and running

Log into your laptop via SSH and run these commands:

sudo apt update && sudo apt install git -y
git clone https://github.com/dw-0/kiauh.git
./kiauh/kiauh.sh
